1. 배열
	1-1. 정의
		크기가 정해진, 데이터를 물리적으로 연속적인 메모리에 저장하는 방식의 자료구조
	1-2. 장단점
		인덱싱을 이용하여 데이터 추가, 삭제, 접근이 O(1)로 매우 빠르나 탐색은 O(n)의 시간 복잡도를 갖는다.
		크기가 정해져있고 물리적으로 연속적인 메모리 공간을 필요로 하기 때문에 메모리를 비효율적으로 사용하게 된다.

2. 선형리스트
	2-1. 정의
		크기가 정해지지 않은, 데이터를 물리적으로 연속적인 메모리에 저장하는 방식의 자료구조.
	2-2. 장단점
		인덱싱을 이용하여 데이터 접근이 O(1)로 매우 빠르고 탐색에는 O(n)의 시간 복잡도를 가지나, 데이터 추가 및 삭제 시 연속된 물리 주소 유지를 위해 많은 시간이 소모된다.
		물리적으로 연속된 메모리 공간을 필요로 하기 때문에 메모리를 비효율적으로 사용하게 된다.
	2-3. 구현
		배열을 이용하여 구현한다.
		임의의 크기의 배열을 생성하고, 데이터가 추가되어 배열이 가득 차게 되면 더 큰 크기의 배열을 생성하여 데이터를 옮기는 방식으로 구현할 수 있다.
