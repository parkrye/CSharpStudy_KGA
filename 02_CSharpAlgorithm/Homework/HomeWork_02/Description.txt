연결 리스트
1. 정의
	연결 리스트란 데이터와 다른 노드의 주소를 가진 노드를 이용하여, 물리적으로 불연속적인 메모리 공간에 연속적인 데이터 집합을 저장할 수 있는 자료구조.
2. 장단점
	메모리 공간을 연속적으로 사용할 필요가 없기 때문에 효율적인 메모리 사용이 가능하다.
	또한 중간에 데이터를 추가하거나 삭제할 경우 앞뒤 노드가 가지고 있는 주소을 수정하여 빠르게 추가 ,삭제가 가능하다.
	다만 인덱싱이 불가능하여 데이터를 검색할 때 O(n)의 시간 복잡도를 갖는다.
3. 구현
	앞 노드의 주소, 뒷 노드의 주소, 데이터를 가진 노드를 이용하여 구현한다.
